# 人工确认流程

## 触发条件

以下情况需要人工确认：

1. **新建 MOC**
   - 当发现新的大主题需要建立知识地图时
   - 需要确认主题范围和命名

2. **核心论文归类**
   - P0 级论文的阅读级别确认
   - 核心方向的判定

3. **认知冲突**
   - 新论文与旧知识冲突时
   - 需要决定保留、修订或废弃旧理解

4. **研究方向调整**
   - 发现需要改变研究焦点时
   - 优先级关键词需要更新时

5. **其他关键决策**
   - 知识库结构变更
   - 重要记忆的验证/推翻

## 流程

```
1. Agent 发现需要确认的事项
   ↓
2. 创建确认请求到 0-Inbox/
   ↓
3. 通知用户（如配置了通知）
   ↓
4. 用户查看并做出决定
   ↓
5. Agent 根据决定执行后续操作
   ↓
6. 记录确认结果到 4-Outputs/Confirmation-Records/
```

## 确认请求格式

使用 Template-Confirmation-Request.md 模板。

## 用户选项

### 同意
- Agent 按建议方案执行
- 记录为 "approved"

### 修改
- 用户提出调整意见
- Agent 根据修改后的方案执行
- 记录为 "modified"

### 拒绝
- 不采纳建议
- Agent 记录原因并寻找替代方案
- 记录为 "rejected"

## 响应时限

- **紧急**（认知冲突影响当前工作）: 24 小时内
- **普通**（新建 MOC、论文归类）: 3 天内
- **低优先级**: 1 周内

## 超时处理

如果用户未及时响应：
1. Agent 在下次交互时提醒
2. 如仍无响应，使用保守方案（如暂缓执行、保持现状）
3. 记录为 "pending-timeout"
