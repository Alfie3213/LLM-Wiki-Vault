# ScholarSkill 长任务处理说明

版本: v1.0.0
最后更新: 2026-03-18

## 背景

ScholarSkill 原设计使用 OpenClaw 的 `durable-task-runner` 进行长任务编排。在 Lingma 环境中，长任务通过会话内顺序执行完成。

## 场景对比

| 场景 | OpenClaw + Durable | Lingma 方案 |
| --- | --- | --- |
| L3 精读 (2.5 小时) | 中断后可恢复 | 建议分阶段执行，或单次会话完成 |
| 批量阅读 (10+ 篇) | 实时进度显示 | 分批处理，每批 3-5 篇 |
| 周巩固 (60-90 分钟) | 状态持久化 | 单次会话内完成 |
| 崩溃恢复 | 自动恢复 | 手动重新执行 |
| 任务编排 | 并行 + 依赖管理 | 顺序执行 |

## Lingma 长任务最佳实践

### L3 精读

建议拆分为多个子任务：

```
阶段 1: 快速浏览 + 优先级评估
阶段 2: 方法精读（1 小时）
阶段 3: 实验分析 + 笔记整理（1 小时）
阶段 4: 记忆抽取 + 知识关联（30 分钟）
```

### 批量阅读

分批处理：

```
批次 1: L1 筛选全部论文（5 分钟 × N 篇）
批次 2: P0 论文 L3 阅读
批次 3: P1 论文 L2 阅读（每批 3-5 篇）
批次 4: P2 论文存档
```

### 进度追踪

在 Lingma 中，可以通过以下方式追踪进度：

1. 在 `1-Papers/To-Process/` 中维护待处理列表
2. 每完成一篇，移动到 `1-Papers/By-Topic/`
3. 在 `wiki/log.md` 中记录操作日志

## 任务控制

在 Lingma 中：
- **暂停**: 停止当前操作，稍后继续
- **恢复**: 从上次停止点继续
- **取消**: 终止当前任务

## 性能优化建议

1. **单次会话控制**: 建议单次会话不超过 2 小时
2. **保存中间结果**: 每完成一个阶段就保存笔记
3. **使用日志**: 在 `wiki/log.md` 中记录进度
4. **分阶段确认**: L3 阅读可分 2-3 次会话完成
