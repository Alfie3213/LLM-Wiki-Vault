# [L3 精读] Automatic Parameter Tuning of Self-Driving Vehicles

## 核心问题

**研究动机**: 现代自动驾驶系统的轨迹规划和控制组件包含大量需要针对不同驾驶场景和车辆类型进行调整的参数。手动调参耗时巨大，且难以在不同车辆平台上保持一致性能。本文旨在解决这一工业痛点，提出一种基于有限专家示范数据的自动参数调优方法。

**核心问题**: 如何利用局部优化技术，基于少量带噪声的真实驾驶示范数据，自动调整 MPC 轨迹规划器的代价函数权重，使其闭环行为与专家驾驶风格一致？

**一句话概括**: 本文提出一个基于闭环仿真代价函数的迭代优化框架，比较了梯度下降、无迹卡尔曼滤波和最大似然估计三种局部优化方法在 BMW 真实车道保持场景中的 MPC 参数自动调优效果，证明即使噪声数据下也能显著改善手动调参。

## 核心贡献

1. **闭环仿真代价函数框架**: 不直接比较开环规划输出与示范，而是比较控制器闭环仿真轨迹与专家示范的偏差，更真实地反映控制器实际表现。
2. **三种局部优化方法对比**: 系统比较了 GD、UKF、ML 在相同问题设置下的表现，为工业界参数调优提供了方法选择依据。
3. **tanh 边界变换技巧**: 通过 tanh 将有界参数映射到无界空间，避免了显式约束处理，提高了优化可行性和收敛速度。
4. **真实场景验证**: 在 BMW Aschheim 测试场的真实数据上验证，使用量产车辆传感器数据（含噪声），证明了方法的工业实用性。

## 背景与问题定义

### 问题背景

自动驾驶系统的模块化 sense-plan-act 架构中，轨迹规划器和控制器的参数调优是关键但耗时的环节。现有方法包括：

| 方法类别 | 代表工作 | 局限性（本文视角） |
|----------|----------|-------------------|
| **IRL/IOC** | [Kuderer et al., 2015], [Rosbach et al., 2019] | 计算需求高，需要大量数据或高保真仿真模型 |
| **贝叶斯优化** | [Neumann-Brosig et al., 2020] | 需要试错式的 episodical 学习 |
| **遗传算法/粒子群** | [Du et al., 2016], [Lim et al., 2022] | 全局优化计算量大，或可能过早收敛 |
| **卡尔曼滤波** | [Menner et al., 2023] | 需要精确的系统先验知识和噪声分布 |

本文的核心假设是：**工业界需要的是基于有限现场数据、计算高效、无需大量试错的方法。**

### 形式化定义

**车辆模型** (离散时间):
$$x_{t+1} = f(x_t, u_t)$$
其中 $x_t \in \{x \in \mathbb{R}^{n_x}: x_{min} \leq x \leq x_{max}\}$，$u_t \in \{u \in \mathbb{R}^{n_u}: u_{min} \leq u \leq u_{max}\}$

**待调参的规划器/控制器**:
$$u_t = \pi(x_t; p), \quad p \in \{p \in \mathbb{R}^{n_p}: p_{min} \leq p \leq p_{max}\}$$

参数 $p$ 可以是 PID 增益、MPC 代价函数权重、或神经网络权重。

**专家示范**:
$$r: y_d|_{[0,T]}$$

**调优代价函数**:
$$J(y_d|_{[0,T]}, x|_{[0,T]}) = \sum_{t=0}^{T} \ell(y_{d,t}, x_t)$$

本文使用加权平方误差:
$$J_k = \sum_{t=0}^{T} (x_t - C y_{d,t})^\top Q (x_t - C y_{d,t})$$

其中 $C$ 确保观测性，$Q$ 为权重矩阵。

**关键假设**:
- 系统状态 $x_t$ 可从测量 $y_{d,t}$ 观测（或重构）
- 参数 $p$ 和测量数据 $y_d$ 的选择使得最优调参问题有唯一解
- 代价函数 (3) 对参数 $p$ 可微

### 挑战与难点

1. **非凸性**: 参数优化问题通常非凸，不同初始值收敛到不同局部最优
2. **噪声数据**: 量产车辆传感器数据含噪声，影响调参精度
3. **可微性要求**: 梯度方法要求闭环仿真对参数可微
4. **有界约束**: 参数、状态、输入均有物理边界限制
5. **多目标权衡**: 代价函数各权重之间存在冲突（如小横向偏差 vs 小加速度）

## 方法

### 核心思想

本文的核心洞见是：**将参数调优转化为一个标准优化问题，通过闭环仿真评估参数质量，并利用局部优化技术高效求解。**

框架流程（图 1）:
1. 用当前参数 $p_k$ 运行规划器/控制器闭环仿真
2. 计算仿真轨迹与专家示范的偏差代价 $J_k$
3. 使用优化算法更新参数 $p_{k+1}$
4. 重复直到收敛

### 技术细节

#### tanh 边界变换

这是本文的一个精巧但重要的技术细节。对于任意有界变量 $x_{1,t} \in [x_{1,min}, x_{1,max}]$，定义无界替代变量 $\tilde{x}_{1,t} \in \mathbb{R}$：

$$x_{1,t} = \frac{x_{1,max} - x_{1,min}}{2} \tanh(\tilde{x}_{1,t}) + \frac{x_{1,min} + x_{1,max}}{2}$$

**优势**:
- 确保解在可行范围内（tanh 输出 ∈ [-1, 1]）
- 无需显式处理边界约束
- 减少搜索空间，加速收敛
- 允许纳入实际物理限制

这一技巧在 Auto-tuning Framework [Haoyang Fan, 2018] 中也有使用，但本文将其应用于 MPC 权重调优场景。

#### 梯度下降 (GD)

$$p_{k+1} = p_k - \gamma \nabla_p J_k(x_d|_{[0,T]}, x|_{[0,T]})$$

**实现细节**:
- 梯度通过数值方法计算（前向仿真 + 有限差分）
- 学习率 $\gamma$ 可根据参数范围自适应调整
- 终止条件：最大迭代次数 $K$ 或 $|J_k - J_{k+1}| < \varepsilon$

**特点**: 实现最简单，计算效率最高，但仅利用代价信息，不考虑数据噪声的概率结构。

#### 无迹卡尔曼滤波 (UKF)

将参数增广到系统状态中：$\tilde{x}_t = [x_t^\top, p^\top]^\top$

增广模型：
$$\tilde{x}_{t+1} = \tilde{f}(\tilde{x}_t, u_t) + w_t$$
$$\tilde{y}_t = \tilde{C} \tilde{x}_t + v_t$$

其中 $w_t \sim \mathcal{N}(0, Q_e)$, $v_t \sim \mathcal{N}(0, R_e)$，且
$$Q_e = \text{blkdiag}(Q, \mu_e I_{n_p})$$

**特点**: 
- 同时估计状态和参数
- 利用 sigma 点传播处理非线性
- 最小化代价的近似期望值
- 需要指定过程噪声和测量噪声的协方差

#### 最大似然估计 (ML)

利用增广模型 (6)，将代价函数 (3) 作为似然函数：

$$\hat{p}_k = \arg\max_{p_k} (-\ln J_k(x_d|_{[0,T]}, x|_{[0,T]} | p_k))$$

**特点**:
- 包含数据的概率假设
- 同时估计参数和协方差
- 通过梯度下降求解（同样需要数值梯度）
- 对噪声数据的鲁棒性理论上最好

### 三种方法的比较

| 特性 | GD | UKF | ML |
|------|-----|-----|-----|
| 计算复杂度 | 最低 | 中等 | 较高 |
| 噪声处理 | 无显式处理 | 通过卡尔曼增益 | 通过似然函数 |
| 概率框架 | 无 | 有 | 有 |
| 超参数需求 | 学习率 | 噪声协方差 + 学习率 | 初始协方差 + 学习率 |
| 实现难度 | 最简单 | 中等 | 较复杂 |
| 理论保证 | 局部收敛 | 近似最优估计 | 渐进一致 |

## 实验

### 实验设置

#### 应用场景: 横向轨迹规划（车道保持）

**规划器**: 线性时变约束 MPC，基于 [Gutjahr et al., 2016]

**系统状态**: $x = [d, \theta, \kappa, \dot{\kappa}]^\top$
- $d$: 车辆后轴中心到参考曲线的有符号法向距离
- $\theta$: 车辆朝向角
- $\kappa$: 曲率
- $\dot{\kappa}$: 曲率变化率（作为系统输入 $u$）

**离散时间模型** (采样时间 $T_s = 0.3$s):

$$x_{t+1} = A(v) x_t + B(v) u_t + E(v) z_t$$

其中 $z = \theta_r$ 为参考朝向（扰动输入），$v$ 为车速（时变参数）。

**MPC 代价函数**:
$$\min_{u_{[0,N]}} \sum_{n=0}^{N} ([x_n - x_{r,n}]^\top Q [x_n - x_{r,n}] + u_n^\top R u_n)$$

其中 $Q = \text{diag}(w_d, w_\theta, w_\kappa, w_{\dot{\kappa}})$, $R = w_u$

**待调参数**: $p = [w_d, w_\theta, w_\kappa, w_{\dot{\kappa}}, w_u]^\top$

#### 专家示范数据

- **来源**: BMW Aschheim 测试场人类驾驶员手动驾驶
- **采样率**: 80 ms
- **场景**: 右弯（约 90 km/h）→ 减速 → 左急弯（约 60 km/h）
- **测量状态**: $y_d = [d_d, \theta_d, \kappa_d]^\top$
- **特点**: 驾驶员为舒适略微切线（横向偏差不为 0）；使用量产传感器，数据含噪声；速度非恒定

#### 调优代价函数

$$J = \sum_{t=0}^{T} (\Delta d_t^2 + \Delta \theta_t^2 + \Delta \kappa_t^2 + \Delta \dot{\kappa}_t^2)$$

其中 $\Delta d = d - d_d$, $\Delta \theta = \theta - \theta_d$, $\Delta \kappa = \kappa - \kappa_d$, $\Delta \dot{\kappa} = \dot{\kappa} - 0$

#### 超参数设置

| 参数 | 值 |
|------|-----|
| $T_s$ | 0.3 s |
| $N$ | 20 |
| $x_{min}$ | $[-2, -1000, -0.04, -0.15]^\top$ |
| $x_{max}$ | $[2, 1000, 0.04, 0.15]^\top$ |
| $u_{min/max}$ | $\pm 0.07$ |
| $p_{min}$ | $10^{-6} [1,1,1,1,1]^\top$ |
| $p_{max}$ | $[10^5, 10^5, 10^5, 10^5, 10^{-3}]^\top$ |
| 最大迭代 $K$ | $10^4$ |
| 学习率 $\alpha$ | 0.002 |
| 终止阈值 $\varepsilon$ | 0.001 |
| $Q$ | $\text{diag}([1, 1, 1, 10^{-8}])$ |
| $\mu_e$ | $10^{-8} [1,1,1,1]^\top$ |
| $R_e$ | $\text{diag}([0.005^2, 0.005^2, 0.0039^2, 0.0001])$ |
| $\hat{P}_0$ | $\text{diag}([1,1,1,1,1,1,1,1,1])$ |

**实现工具**: Python, FilterPy (KF), SciPy (ML), CasADi (QP solver)

### 主要结果

#### 参数优化结果 (Table I)

| 参数 | $w_d$ | $w_\theta$ | $w_\kappa$ | $w_{\dot{\kappa}}$ | $w_u$ |
|------|-------|-----------|-----------|------------------|-------|
| $p_0$ (初始) | 3.443 | 0.535 | 0.535 | $0.032 \times 10^{-6}$ | — |
| $p$: GD | 9.59 | 0.536 | 0.535 | $0.032 \times 10^{-6}$ | — |
| $p$: KF | 0.76 | 0.114 | 0.117 | 0.00719 | 0.0019 |
| $p$: ML | 3.93 | 1.64 | 0.01 | 0.01 | $1 \times 10^{-7}$ |

**关键观察**:
- GD 主要调整了 $w_d$（横向偏差权重），其他权重几乎不变
- KF 对所有权重都做了较大幅度调整
- ML 大幅降低了 $w_\kappa$ 和 $w_{\dot{\kappa}}$，同时提高了 $w_\theta$
- 三种方法收敛到**不同的局部最优**，验证了问题的非凸性

#### 轨迹质量对比 (Table II)

| 方法 | $\sum \Delta d_d^2$ | $\sum \Delta \theta_d^2$ | $\sum \Delta \kappa_d^2$ | Cost |
|------|-------------------|----------------------|----------------------|------|
| init (手动) | 34.9 | 0.0289 | $1.31 \times 10^{-4}$ | 34.9 |
| GD | 29.6 | 0.0226 | $1.61 \times 10^{-4}$ | 29.6 |
| KF | 32.0 | 0.0179 | $1.05 \times 10^{-4}$ | 32.0 |
| ML | 28.2 | 0.0229 | $1.71 \times 10^{-4}$ | 28.2 |

**结果分析**:
- **ML 最优**: 整体代价最低 (28.2)，横向偏差最小
- **GD 次之**: 整体代价 29.6，表现接近 ML
- **KF 第三**: 整体代价 32.0，但航向角偏差最小（0.0179）
- 所有方法均显著优于手动调参 (34.9)

#### 时域行为分析 (Fig. 4)

**前段（直线 + 小右弯，0-30s）**:
- 三种方法表现相似，都能较好地跟踪示范轨迹
- 优化后的行为在 20-30s 时间段内很好地平衡了示范轨迹和参考路径

**后段（左急弯，30-80s）**:
- 示范轨迹的横向偏差高达 45 cm（驾驶员切线）
- 优化后的行为横向偏差显著减小，因为 MPC 本身倾向于最小化相对于参考路径的偏差
- ML 和 GD 行为可比，KF 略差

### 分析与洞察

#### 为什么三种方法收敛到不同局部最优？

问题的非凸性源于：
1. **MPC 的隐式非线性**: 即使 MPC 本身是 QP，其解作为参数的函数是非线性的
2. **约束的激活/去激活**: 不同参数导致不同的约束激活模式
3. **时变速度**: 速度变化使系统矩阵 $A(v)$ 时变，增加了复杂性

#### 为什么 ML 表现最好？

1. **概率建模**: ML 显式建模了数据噪声，对量产传感器噪声更鲁棒
2. **协方差估计**: 同时估计参数的不确定性，避免了过度拟合噪声
3. **似然函数的形状**: 对数似然函数可能比原始代价函数具有更好的优化 landscape

#### KF 为什么在某些指标上更好？

尽管整体代价不如 ML 和 GD，KF 在航向角偏差 ($\sum \Delta \theta_d^2$) 上表现最优 (0.0179 vs ML 0.0229)。这可能是因为：
- KF 的状态估计机制更有效地利用了航向角测量信息
- 增广状态空间允许参数根据状态估计误差自适应调整

#### 与 IRL 方法的对比

本文方法与 IRL 方法（如 Drive with Style [Rosbach et al., 2019]）的关键差异：

| 维度 | 本文方法 | IRL (Drive with Style) |
|------|---------|----------------------|
| 学习目标 | 固定结构 MPC 的权重 | 通用规划器的奖励函数 |
| 数据需求 | 少量示范（单场景） | 较多示范（多场景） |
| 计算方式 | 局部优化，高效 | MaxEnt IRL，需采样 |
| 泛化能力 | 针对特定控制器 | 可迁移到不同规划器 |
| 工业适用性 | 高（直接调参） | 中（需重新训练） |

本文方法更适合**工业部署中的控制器微调**，而 IRL 更适合**从数据中发现新的驾驶风格**。

## 关键洞察

1. **闭环仿真优于开环比较**: 本文的闭环仿真框架比直接比较规划器输出与示范更真实，因为它捕捉了控制器在实际闭环中的行为。
2. **非凸性是参数调优的核心挑战**: 三种方法收敛到不同最优的事实表明，参数调优问题的非凸性不容忽视。工业实践中应尝试多种初始值。
3. **概率方法对噪声数据有优势**: ML 在噪声真实数据上的最优表现验证了概率框架的价值，尽管其实现更复杂。
4. **tanh 变换是实用的工程技巧**: 这一简单而有效的边界处理技巧可以广泛应用于有界参数优化问题。
5. **参数调优与 IRL 是互补的**: 参数调优适合已知代价函数结构下的权重优化；IRL 适合发现新的代价函数结构。两者可以组合使用（先用 IRL 发现结构，再用参数调优微调权重）。

## 疑问与验证

### 疑问

1. **多场景一致性**: 本文仅在单一场景（右弯→左急弯）上验证。同一组优化后的参数在其他场景（如换道、超车、城市道路）上是否仍然表现良好？
2. **非凸性的系统性分析**: 作者提到问题非凸，但没有量化分析代价函数 landscape 的复杂度（如局部最优的数量、 basin of attraction 的大小）。
3. **GD 的权重不变现象**: GD 对 $w_\theta, w_\kappa, w_{\dot{\kappa}}$ 几乎不变，仅调整 $w_d$。这是否意味着梯度在这些方向上几乎为 0？如果是，是否意味着这些权重对轨迹质量不敏感？
4. **与 Apollo auto-tuning 框架的关系**: [Haoyang Fan, 2018] 提出的 RC-IRL 框架也用于自动驾驶参数调优，本文方法与 RC-IRL 的排序条件方法有何异同？
5. **实时性**: 参数调优是在离线完成的，但工业实践中是否可能需要在线自适应调参？

### 验证计划

1. 在多个驾驶场景上测试同一组优化参数，评估泛化能力
2. 可视化代价函数在参数空间中的 landscape，量化局部最优数量
3. 分析 GD 的梯度方向，验证哪些参数方向对代价函数最敏感
4. 将本文方法与 RC-IRL 在相同场景上进行对比实验
5. 探索在线参数自适应的可能性（如根据场景类型切换参数集）

## 批判性分析

### 优势

1. **工业实用性强**: 方法简单、计算高效、易于在现有 MPC 框架中集成
2. **真实数据验证**: 使用 BMW 测试场的真实数据（含噪声），证明了方法的鲁棒性
3. **系统对比**: 三种方法的对比为工业界提供了清晰的选择依据
4. **边界处理优雅**: tanh 变换简洁有效地处理了有界约束
5. **开源工具链**: 使用 Python + CasADi + FilterPy + SciPy，便于复现

### 局限性

1. **单场景验证**: 仅在一种驾驶场景（车道保持）上验证，泛化能力未充分证明
2. **局部最优问题**: 未提供处理多局部最优的策略（如多初始值、全局优化初始化）
3. **参数维度有限**: 仅调优 5 个 MPC 权重，未涉及规划 horizon、采样时间等更复杂的参数
4. **计算开销未报告**: 未报告三种方法的实际计算时间对比
5. **与专家手动调参的公平性**: 初始参数 $p_0$ 是"手动调参"的结果，但手动调参过程未详细描述，可能不是最优手动调参
6. **代价函数设计**: 调优代价函数 (10) 的选择有一定任意性（等权重平方和），不同权重可能改变优化结果

### 可借鉴之处

1. **闭环仿真评估框架**: 将参数调优转化为闭环仿真+优化的范式可以推广到其他控制器类型
2. **tanh 边界变换**: 可以广泛应用于任何有界参数优化问题
3. **多方法对比实验设计**: 系统对比多种优化方法的实验设计思路

### 可改进方向

1. **多场景联合优化**: 同时在多个场景上优化，提高泛化能力（作者在 Future Work 中提到）
2. **全局+局部混合优化**: 先用全局优化（如贝叶斯优化）找到好的初始值，再用局部优化精调
3. **敏感性分析**: 量化每个参数对轨迹质量的影响，指导参数选择
4. **在线自适应**: 根据场景识别结果动态切换参数集
5. **更复杂的参数**: 将规划 horizon、采样时间等纳入调优范围

## 反思

### 成功之处

本文在简洁的框架内解决了一个实际的工业问题。闭环仿真+局部优化的思路非常直接，但正是这种直接性使其具有高度的工程实用性。三种方法的系统对比也提供了有价值的实证 insights——ML 的最优表现说明在噪声数据场景下，概率框架确实有优势。

### 遇到的问题

1. 单场景验证的局限性：虽然证明了概念，但工业界更关心多场景一致性
2. 非凸性未充分处理：作者承认问题非凸，但未提供有效的应对策略
3. 调优代价函数的设计：等权重平方和可能不是最优选择

### 改进方案

1. 增加多场景验证实验
2. 引入多初始值策略或全局优化初始化
3. 探索调优代价函数的加权设计（如根据重要性加权不同状态偏差）

### 关键收获

1. **工程简洁性**: 最简单的方法往往最实用。本文没有使用复杂的深度学习或 IRL，而是用基本的优化技术解决了实际问题。
2. **闭环评估的重要性**: 评估控制器参数时必须使用闭环仿真，开环评估会丢失关键动态信息。
3. **概率方法的价值**: 即使实现更复杂，概率方法（ML）在噪声数据上的优势值得投入。

   - evidence: 本文在 BMW 真实车道保持场景中，使用闭环仿真+局部优化将 MPC 代价函数权重从手动调参的 Cost=34.9 优化到 ML 的 Cost=28.2（提升 19.2%）

2. **Tanh-Bound-Transform**
   - claim: tanh 变换是一种简洁有效的有界参数优化技巧，通过将有界变量映射到无界空间，避免了显式约束处理并提高了优化可行性
   - evidence: 本文在 5 维 MPC 权重调优中使用 tanh 变换，确保参数始终在物理可行范围内，同时允许使用无界优化器
   - scope: 有界参数优化问题
   - related_to: [[AutomaticParameterTuning]], [[OptimalControl]]
   - confidence: high
   - action: 在任何有界参数优化问题中尝试使用 tanh 变换替代显式约束
   - memory_tier: core

#### 检索型记忆

3. **GD-UKF-ML-Comparison**
   - claim: 在自动驾驶 MPC 参数调优中，最大似然估计（ML）在噪声真实数据上表现最优，无迹卡尔曼滤波（UKF）次之，梯度下降（GD）第三，但三者均显著优于手动调参
   - evidence: BMW 测试场实验中，ML Cost=28.2, GD=29.6, KF=32.0, 手动=34.9
   - scope: 自动驾驶控制器参数调优的方法选择
   - related_to: [[AutomaticParameterTuning]], [[MaximumEntropyIRL]]
   - confidence: high
   - action: 在噪声数据场景下优先尝试 ML 方法，在计算资源受限时尝试 GD
   - memory_tier: retrieval

4. **Param-Tuning-vs-IRL**
   - claim: 参数调优与逆强化学习是互补的——参数调优适合已知代价函数结构下的权重优化（工业微调），IRL 适合发现新的代价函数结构（研究探索）
   - evidence: 本文明确对比了与 IRL 方法的差异，指出 IRL 计算需求高、需要大量数据，而参数调优更适合有限现场数据的工业场景
   - scope: 自动驾驶代价函数学习方法的选择
   - related_to: [[InverseReinforcementLearning]], [[AutomaticParameterTuning]], [[DriveIRL]]
   - confidence: high
   - action: 在工业部署阶段使用参数调优微调控制器，在研究阶段使用 IRL 探索驾驶风格
   - memory_tier: retrieval

5. **Nonconvexity-Local-Optima**
   - claim: 自动驾驶控制器参数调优问题具有显著的非凸性，不同优化方法从同一初始值出发可能收敛到不同的局部最优
   - evidence: 本文实验中 GD、UKF、ML 从相同的 $p_0$ 出发，收敛到完全不同的参数值（如 $w_d$: GD=9.59, KF=0.76, ML=3.93）
   - scope: 参数优化问题的收敛性分析
   - related_to: [[AutomaticParameterTuning]]
   - confidence: high
   - action: 在参数调优实践中使用多初始值策略，并比较不同局部最优的表现
   - memory_tier: retrieval

   - evidence: 本文详细描述了 BMW 的横向 MPC 实现，采样时间 0.3s，规划 horizon N=20，使用 CasADi QP 求解器
   - scope: 工业界自动驾驶横向控制实现
   - related_to: [[AutomaticParameterTuning]], [[MPCbasedImitationLearning]]
   - confidence: high
   - action: 在设计横向 MPC 时参考 BMW 的 5 维权重结构

### Episodic Memories（阅读洞察，2 条）

1. **GD-Weight-Invariance-Puzzle**
   - episode: 发现 GD 仅调整了 $w_d$ 而其他权重几乎不变时的困惑
   - initial_understanding: 以为 GD 会均匀调整所有权重
   - confusion: 为什么 $w_\theta, w_\kappa$ 几乎不变？
   - resolution_path: 重新分析问题结构后理解——代价函数 (10) 中各状态的权重可能具有不同的敏感性。横向偏差 $d$ 在场景中的动态范围最大（45 cm），因此对代价函数的贡献最大，梯度也最大。其他状态的贡献相对较小，梯度接近 0。
   - verified: partially（需要梯度可视化验证）
   - future_hint: 在参数调优前应先进行敏感性分析，识别对代价影响最大的参数方向

2. **ML-Superiority-Insight**
   - episode: 最初认为三种方法表现应该相近，因为都是局部优化
   - initial_understanding: GD 最简单，应该足够好
   - confusion: 为什么 ML 在噪声数据上明显更好？
   - resolution_path: 仔细阅读 ML 的公式 (7) 后理解——ML 通过似然函数显式建模了数据噪声的概率结构，而 GD 完全忽略了噪声。在真实传感器数据（含噪声）上，这种概率建模带来了实质性的性能提升。
   - verified: true
   - future_hint: 在处理真实传感器数据时，优先考虑具有概率建模能力的方法

### Procedural Memories（程序规则，3 条）

1. **Closed-Loop-Eval-Rule**
   - rule: 评估自动驾驶控制器参数时，必须使用闭环仿真而非开环规划输出
   - applies_to: 任何自动驾驶控制器/规划器的参数调优或评估
   - why_it_works: 闭环仿真捕捉了控制器与车辆动力学之间的耦合效应，更接近实际部署表现
   - when_not_to_use: 纯开环规划器（无闭环反馈）的评估
   - confidence: high
   - action: 在实现参数调优系统时，确保评估阶段使用完整的闭环仿真

2. **Tanh-Transform-Rule**
   - rule: 当优化问题中的变量具有物理边界时，使用 tanh 变换将其映射到无界空间
   - applies_to: 任何有界参数/状态/输入的优化问题
   - why_it_works: tanh 输出天然有界，避免了显式约束处理，同时保持可微性
   - when_not_to_use: 变量无界或边界不重要的场景
   - confidence: high
   - action: 在实现 MPC 权重调优或其他有界参数优化时，将 tanh 变换作为默认预处理步骤

3. **Probabilistic-For-Noisy-Data-Rule**
   - rule: 当使用真实传感器数据进行优化时，优先选择具有显式概率建模的方法（如 ML、UKF）而非纯确定性方法（如 GD）
   - applies_to: 任何基于真实传感器数据的学习或优化任务
   - why_it_works: 真实数据包含不可忽略的系统噪声和测量误差，概率方法通过建模这些不确定性获得更鲁棒的估计
   - when_not_to_use: 数据质量极高（如仿真完美数据）或计算资源极度受限
   - confidence: high
   - action: 在基于真实车辆数据的参数学习中，将 ML 或 UKF 作为首选方法
