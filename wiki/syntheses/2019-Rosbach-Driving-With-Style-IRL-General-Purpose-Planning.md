# [L3 精读] Driving with Style: Inverse Reinforcement Learning in General-Purpose Planning for Automated Driving

## 核心问题

**研究动机**: 自动驾驶中的行为规划和运动规划传统上采用分层架构解耦处理，行为规划器用预定义模板指令局部运动规划器。但城市环境中场景复杂度高，不可预见情况频繁发生，行为规划器难以匹配预定义模板。近年来出现的通用型规划器（General-Purpose Planner）将行为与运动规划整合，通过单一奖励函数实现行为感知运动规划。然而，这种通用型规划器面临两个核心挑战：一是奖励函数必须将复杂特征空间映射为奖励值，二是该函数需由专家手动调参，过程繁琐且难以在所有场景中编码类人驾驶风格。

**核心问题**: 如何自动化通用型规划器的奖励函数调参过程，使学习得到的奖励函数能够编码类人驾驶风格，且效果超越运动规划专家的手动调参？

**一句话概括**: 提出将路径积分最大熵逆强化学习（Path Integral MaxEnt IRL）嵌入基于 GPU 并行采样的通用型规划器，利用规划器自身的图表示近似配分函数，在大众原型车上验证学习奖励超越专家手动调参（EVD 降低 63%-67%，ED 降低 44%-54%）。

## 核心贡献

1. **IRL 与通用型规划器的集成**: 将最大熵 IRL 与模型预测通用型规划器结合，在单一框架中实现类人驾驶风格的线性奖励函数编码
2. **路径积分 MaxEnt IRL 公式化**: 针对高维连续空间中的驾驶风格优化，提出利用规划器图表示近似配分函数和期望特征计数的方法，避免了传统状态访问频率计算的维度灾难
3. **超越专家调参**: 通过多种性能指标和真实世界测试，证明自动化奖励学习方法优于运动规划专家的手动调参
4. **环境特定奖励生成**: 自动化调参过程可为不同驾驶环境生成优化后的奖励函数，扩展线性奖励函数的泛化能力

## 背景与问题定义

### 传统分层规划的局限

传统自动驾驶规划架构分为三个层级：
1. **战略层**: 路径规划（导航信息）
2. **战术/行为层**: 行为规划（变道、跟车、紧急制动等预定义动作）
3. **操作层**: 局部运动规划（参考轨迹生成）

**核心问题**: 行为规划缺乏运动约束的充分知识，导致：
- 动作可能因高估能力而不可行
- 动作可能因低估能力而被错误丢弃
- 复杂和不可预见场景中难以匹配预定义模板

### 通用型规划器的兴起

[McNaughton, 2011] 提出并行实时规划方法：
- 高分辨率动作采样，满足运动学约束
- 动作序列可表示复杂动作
- 单一奖励函数，可由行为规划器在线调整
- 避免了分层架构的协调复杂性

**遗留问题**: 奖励函数的手动调参和维护仍然繁琐，且调参过程本身超出 [McNaughton] 的工作范围。

### 奖励函数特征空间

- [Heinrich et al., 2016] 提出基于模型的策略，将传感器覆盖范围纳入相关环境信息以优化车辆未来姿态
- [Gu et al., 2016] 从大量采样策略中推导出战术特征
- **空白**: 通用型规划器的自动化奖励函数调参过程讨论甚少

### 形式化定义

**MDP 框架**: $\{S, A, T, R, \gamma\}$
- $S$: 状态集
- $A$: 动作集（连续动作）
- $T(s, a, s')$: 转移函数
- $R$: 奖励函数
- $\gamma$: 折扣因子

**环境模型 $M$**: 返回特征向量 $f$ 和执行动作 $a$ 后的结果状态 $s'$

**线性奖励函数**:
$$R(s, a) = \sum_{i \in K} \theta_i f_i(s, a)$$

**策略 $\pi$**: 时间连续转移 $T$ 的序列

**特征路径积分**:
$$f_i^\pi = \int_t^{t'} f_i(s_t, a_t) dt$$

通过迭代执行采样的状态-动作集 $A_s$ 在环境模型 $M$ 中近似。

**策略价值**:
$$V^\pi = \int_t^{t'} R(s_t, a_t) dt$$

**最优策略**:
$$\pi^* = \arg \max_\pi V^\pi$$

**人类示范**: 车辆里程计记录 $\zeta$，投影到状态-动作空间形成示范策略 $\pi_D$

**规划算法输出**: 有限策略集 $\Pi$，最终策略 $\pi_S$ 满足模型约束

## 方法详解

### A. 通用型规划器

基于 [Heinrich et al., 2016/2018] 的自动驾驶规划算法，使用 GPU 并行性进行动作采样。

**搜索算法**（Algo. 1）:

```
算法 1: General-Purpose Planner
输入: 规划时域 H, 模型 M, 奖励函数 R, 折扣因子 γ
输出: 策略集 Π, 规划解 π_S

1. for t in H do
2.   S_t ← get set of states
3.   for all s ∈ S_t do
4.     A_s ← sample set of actions
5.     for all a ∈ A_s do
6.       execute action a in M(s, a)
7.       observe resultant state s'
8.       observe resultant transition T
9.       observe resultant features f(s, a)
10.      construct labels c(T)
11.      R(s, a) ← Σ_{i∈K} θ_i f_i(s, a)
12.      V(s') ← V(s) + γ^t R(s, a)
13.   S_{t+1} ← prune S_t
14. Π ← get policies in S, A
15. π_S ← select model-based from Π
```

**动作表示**:
- 纵向动作：速度曲线，最高五阶多项式
- 横向动作：转角曲线，最高三阶多项式
- 系数由 actor-friendly 连续性约束推导

**剪枝操作**: 基于价值 $V(s)$、标签 $c$ 和可达集属性终止冗余状态
- 限制状态空间指数增长
- 保证策略集的行为多样性

**算法本质**: 类似并行广度优先搜索和前向价值迭代

### B. 逆强化学习

**目标**: 找到奖励函数权重 $\theta$，使最优策略 $\pi^*$ 至少与示范策略 $\pi_D$ 一样好：
$$V(\pi^*) \geq V(\pi_D)$$

**歧义性问题**: 给定最优策略，多个奖励函数可能导致相同的最优策略 [Ng & Russell, 2000]。

**最大熵 IRL**: 通过最大化策略分布的熵来解决歧义性，避免偏差 [Ziebart et al., 2008]。

**传统梯度计算的局限**: 状态访问频率的计算类似于 RL 中的反向价值迭代，在高维连续空间中因维度灾难而不可行。

### C. 路径积分最大熵 IRL

**对数似然最大化**:
$$\theta^* = \arg \max_\theta L(\theta) = \arg \max_\theta \sum_{\pi_D \in \Pi_D} \ln p(\pi_D | \theta)$$

其中：
$$p(\pi_D | \theta) = \frac{1}{Z} \exp(\theta^T f^{\pi_D})$$

配分函数：
$$Z = \sum_{\pi \in \Pi} \exp(\theta^T f^\pi)$$

**特征匹配约束**:
$$\forall i \in 1, ..., k: \sum_{\pi \in \Pi} p(\pi | \theta) f_i^\pi = \frac{1}{m} \sum_{\pi_D \in \Pi_D} f_i^{\pi_D} = \hat{f}_i^{\Pi_D}$$

**梯度**:
$$\nabla L(\theta) = \sum_{\pi \in \Pi} p(\pi | \theta) f^\pi - \hat{f}^{\Pi_D}$$

**配分函数近似的核心洞察**:
- 通用型规划器的 GPU 并行动作采样产生大量策略（平均约 4000 条）
- 剪枝操作确保多行为探索的可达集
- 因此规划器自身的策略集可用于近似配分函数（类似 MCMC）
- **无需额外采样模块**，直接复用规划器的搜索图

**训练流程**:
1. 人类驾驶示范时，记录环境特征和里程计
2. 每个规划周期，近似策略集的特征路径积分向量 $f^\Pi$，存入回放缓冲区
3. 通过投影度量将策略与示范里程计关联
4. 训练时查询低投影度量值的示范策略
5. 使用经验回放，随机抽取小批量经验进行多次训练
6. 更新奖励函数后，规划器配置更新，具有类似人类特征的策略获得更高价值

### D. 投影度量

**度量定义**:
$$d(\zeta, \pi) = \int_t^{t'} \gamma^t ||\zeta_t - \pi_t|| dt$$

基于状态空间的几何属性：
- 纵向和横向的欧氏距离
- 偏航角的平方差
- 规划时域上的折扣因子 $\gamma^t$

**投影策略 $\pi_D$**: 具有最小折扣距离的示范策略

**三大优势**:
1. **约束验证**: 投影轨迹包含规划器所有约束，若度量超过阈值，则示范超出车辆能力限制，不可作为有效示范
2. **直观评估**: 基于与里程计的几何接近度评估驾驶风格
3. **数据增强**: 放宽约束，使多个规划器策略可作为示范，增加示范数量

### E. 奖励特征表示

**12 维奖励特征**（$K = 12$）:

**运动特征（7 个）**:
| 特征 | 描述 |
|------|------|
| 速度偏差 | 目标速度与策略速度之差 |
| 纵向加速度 | 纵向动作加速度值 |
| 纵向急动度 | 纵向动作急动度值 |
| 横向加速度 | 横向动作加速度值 |
| 横向急动度 | 横向动作急动度值 |
| 末端方向 | 策略末端朝向驾驶方向的角度 |
| 蠕行抑制 | 在应完全停止的情况下抑制极慢纵向运动 |

**基础设施特征（5 个）**:
| 特征 | 描述 |
|------|------|
| 车道中心接近度 | 与车道中心的距离 |
| 路缘接近度 | 与道路边缘的距离 |
| 车道代价 | 不同车道的代价势能 |
| 方向 | 行驶方向 |
| 冲突区域 | 冲突区域（如斑马线停车） |

**特征归一化**: 感知系统提供归一化的特征图

**折扣因子**: 由运动规划专家手动定义，不优化

## 实验分析

### 数据收集

- **车辆**: 大众集团原型车
- **环境**: 静态环境（禁用目标识别和意图预测）
- **数据**: 感知系统特征 + 人类驾驶员里程计
- **训练方式**: 回放记录数据，每个 MPC 规划周期后将车辆位置重置为人类示范的里程计位置

### 训练评估

**训练设置**:
- 两种道路段类型：直线段（straight）和弯曲段（curvy）
- 每种段两次驾驶：随机初始化奖励函数 vs 专家调参奖励函数
- 四种学习奖励函数：IRLCurve(random)、IRLCurve(expert)、IRLStraight(random)、IRLStraight(expert)

**评估指标 1 — 期望价值差（EVD）**:
$$E[V(\Pi)] - E[V(\Pi_D)] = \sum_{\pi \in \Pi} p(\pi | \theta) V(\pi) - \sum_{\pi_D \in \Pi_D} p(\pi_D | \theta) V(\pi_D)$$

- 初始差异：直线段和弯曲段分别为 30% 和 19%
- 训练后 EVD 降低：弯曲段 67%，直线段 63%

**评估指标 2 — 期望距离（ED）**:
$$E[d(\zeta; \Pi)] = \sum_{\pi \in \Pi} p(\pi | \theta) d(\zeta, \pi)$$

- 训练后 ED 降低：弯曲段 54%，直线段 44%

**关键发现**:
- (a) 投影度量依赖于奖励函数质量
- (b) 改进的奖励函数导致改进的动作采样，从而产生更好的示范
- (c) 无需先验知识即可学习：即使随机初始化也能生成有效回放缓冲区并训练
- (d) 不合适的奖励函数在训练中改善更显著

### 驾驶风格评估

**测试设置**:
- 在专用测试路段上独立回放
- 多次驾驶以生成代表性统计
- 对比奖励函数：Random、Expert tuned、IRL(random)、IRL(expert)

**投影度量分布**（最优策略与人工驾驶里程计的距离）:
- 直道段：944 个规划周期，200 bins，bin size 0.001
- 弯曲段：369 个规划周期

**距离改善**:
| 方法 | 直道段改善 | 弯曲段改善 |
|------|-----------|-----------|
| IRLStraight(random) | **73%** | - |
| IRLCurve(random) | - | **43%** |
| IRLStraight(expert) | 22% | - |
| IRLCurve(expert) | - | 4% |

**关键发现**:
- 学习奖励函数在所有段上均改善驾驶风格，即使随机初始化
- 直道段学习效果显著强于弯曲段（更容易的学习任务）
- 专家调参初始化在平均距离上改善不大，但在规划周期间的距离方差显著降低
- 方差降低表明学习奖励函数在规划周期间具有更高的稳定性

**期望价值与距离的关系**:
- 总体趋势：期望价值与期望距离呈反比关系
- 学习奖励函数的期望距离低于专家和随机奖励函数
- 学习奖励函数在期望距离增加时期望价值下降更快
- 这确保了学习奖励函数在策略评估中引入高度偏差，使人類示范行为更受偏好

### 特征权重分析

奖励权重与特征值的偏好成反比：
- 权重越高 → 对该特征值的惩罚越大 → 策略越避免该特征
- 学习得到的权重可与专家权重对比，进行可解释性分析

## 可借鉴之处

1. **规划器内嵌 IRL**: 将 IRL 直接嵌入规划器的图构建过程，利用规划器自身的搜索能力近似配分函数，无需独立采样模块
2. **经验回放用于 IRL**: 将 RL 中的经验回放机制应用于 IRL，通过存储规划周期的特征向量实现高效多次训练
3. **投影度量三重作用**: 同时作为约束验证器、风格评估器和数据增强器
4. **专家初始化 vs 随机初始化**: 两种初始化策略的对比揭示了不同场景的学习难度和收敛特性

## 可改进方向

1. **动态奖励更新**: 当前训练后奖励函数固定，未来可研究动态更新以适应环境变化
2. **动态环境扩展**: 当前仅在静态环境中验证，未考虑动态障碍物和交互场景
3. **折扣因子优化**: 折扣因子由专家手动设定，未纳入学习过程
4. **更复杂特征**: 当前仅 12 维线性特征，可扩展至非线性或深度特征
5. **多驾驶员风格**: 当前针对单一驾驶风格，未考虑个体多样性
6. **真实世界闭环验证**: 当前为开环回放验证，未进行真实闭环自动驾驶测试

## 批判性分析

### 优势

1. **工业落地导向**: 方法直接应用于大众集团原型车，具有实际工程价值
2. **问题定位精准**: 直击通用型规划器"奖励调参繁琐"的核心痛点
3. **方法简洁优雅**: 利用规划器自身特性解决 IRL 的维度灾难，无需复杂额外模块
4. **实验验证充分**: 同时包含训练收敛分析、驾驶风格评估、期望价值-距离关系分析
5. **可解释性强**: 线性奖励函数的权重可直接解读为对驾驶行为的偏好

### 局限

1. **静态环境限制**: 禁用目标识别和意图预测，仅在静态环境中验证，结果难以推广到真实城市驾驶
2. **开环验证**: 测试为回放模式（每个周期重置到人类示范位置），非真实闭环驾驶
3. **样本效率未报告**: 未明确报告训练所需的示范数据量（距离/时间）
4. **GCL/深度方法缺失**: 未与 GCL 等深度 IRL 方法对比，无法评估在更大数据量下的相对性能
5. **特征工程依赖**: 12 维特征由专家定义，自动化特征学习未探索
6. **弯曲段学习效果有限**: 专家初始化在弯曲段仅改善 4%，说明复杂场景的学习仍具挑战

### 与相关工作的关系

- **vs SMIRL [Wu & Sun, 2020]**: 两者都解决连续域 MaxEnt IRL 的配分函数估计问题。SMIRL 通过独立采样模块（基于运动规划先验）生成样本；Rosbach 等则直接利用规划器自身的 GPU 并行搜索图。SMIRL 更通用（可适配不同规划器），Rosbach 的方法更紧密集成（但依赖特定规划器架构）。
- **vs Opt-IRL [Kuderer et al., 2015]**: Opt-IRL 用最优轨迹代表期望特征，每轮需重新求解优化；Rosbach 等利用规划器预计算的策略集，训练时仅需梯度更新。
- **vs CIOC [Levine & Koltun, 2012]**: CIOC 要求示范最优且需 Hessian 计算；Rosbach 等兼容次优示范且无需二阶信息。
- **vs McNaughton [2011]**: 采用 McNaughton 的通用型规划框架，但填补了其未涉及的奖励调参自动化问题。

## 反思

### 成功之处

- 清晰地识别了通用型规划器的实际痛点：奖励函数的手动调参繁琐且难以在所有场景中保持一致
- 方法设计巧妙：不添加独立模块，而是复用规划器自身的计算能力（GPU 并行搜索图）来近似 IRL 所需的配分函数
- 实验设计全面：同时评估训练收敛（EVD）、风格模仿（ED）和价值-距离关系

### 遇到的问题

- 为何弯曲段的学习效果（尤其是专家初始化）显著弱于直道段？
  - 弯曲段需要更复杂的横向控制和末端方向调整
  - 特征空间更复杂，线性奖励函数的表达能力可能不足
  - 专家初始化已经较好，改善空间有限
- 静态环境验证的局限性是否削弱了方法的实际价值？
  - 作者明确说明这是"this stage of the work"
  - 动态扩展是未来方向

### 改进方案

1. 在动态环境中验证（加入行人、车辆交互）
2. 与 SMIRL 等方法的直接对比实验
3. 探索非线性奖励函数或深度特征表示
4. 将折扣因子纳入优化目标
5. 进行真实闭环自动驾驶测试（而非回放）

### 关键收获

1. **IRL 的配分函数估计可以利用规划器自身能力**: 不局限于独立采样模块，规划器的搜索过程本身就是对状态-动作空间的探索
2. **经验回放机制可跨领域复用**: RL 中的经验回放不仅适用于策略学习，也可用于奖励学习
3. **投影度量设计是连接人类示范与规划器的关键**: 好的投影度量需同时满足几何直观性、约束验证能力和数据增强能力
4. **初始化策略影响学习效果**: 随机初始化在大改善空间中效果显著，专家初始化在稳定方差上更有优势

### Semantic Memories（知识升级，5 条）

#### 常驻记忆块

1. **planner-embedded-IRL**
   - claim: 通用型规划器的 GPU 并行搜索图可直接用于近似 MaxEnt IRL 的配分函数，无需独立采样模块
   - evidence: Rosbach 等在大众原型车上实现，利用规划器平均 4000 条策略集近似配分函数，EVD 降低 63%-67%，ED 降低 44%-54%
   - scope: 基于搜索的自动驾驶规划器中的 IRL 应用
   - related_to: [[GeneralPurposePlanner]], [[SMIRL]], [[MaximumEntropyIRL]], [[Opt-IRL]]
   - confidence: high
   - action: 在使用基于 GPU 并行搜索的规划器时，优先复用规划器自身的策略集进行 IRL，而非构建独立采样器
   - memory_tier: core

2. **projection-metric-three-roles**
   - claim: 投影度量在基于搜索的 IRL 中同时承担约束验证、风格评估和数据增强三种角色
   - evidence: Rosbach 等设计的度量 $d(\zeta, \pi) = \int \gamma^t ||\zeta_t - \pi_t|| dt$ 包含几何距离、偏航角差和折扣因子，超过阈值可过滤无效示范，放宽阈值可增加示范数量
   - scope: 基于搜索的规划器 IRL
   - related_to: [[GeneralPurposePlanner]], [[摘要-driving-with-style-irl-general-purpose-planning]]
   - confidence: high
   - action: 设计基于搜索的规划器 IRL 时，投影度量应同时考虑约束验证、风格评估和数据增强需求
   - memory_tier: core

#### 检索型记忆

3. **experience-replay-IRL**
   - claim: 经验回放机制可应用于 IRL，通过存储规划周期的特征路径积分实现高效多次训练
   - evidence: Rosbach 等将每个 MPC 周期的特征向量 $f^\Pi$ 存入回放缓冲区，训练时随机抽取小批量
   - scope: 需要多次遍历数据的 IRL 方法
   - related_to: [[GeneralPurposePlanner]], [[MaximumEntropyIRL]]
   - confidence: medium
   - action: 在数据收集成本高的 IRL 场景中，使用经验回放提高样本利用效率
   - memory_tier: retrieval

4. **linear-reward-expert-vs-learned**
   - claim: 在通用型规划器的线性奖励函数中，学习得到的权重可以在特定场景超越专家手动调参，但复杂场景（弯曲段）的改善空间有限
   - evidence: 直道段随机初始化 IRL 改善 73%，弯曲段仅 43%；专家初始化弯曲段仅 4%
   - scope: 自动驾驶奖励函数设计
   - related_to: [[GeneralPurposePlanner]], [[InverseReinforcementLearning]]
   - confidence: medium
   - action: 对简单场景可完全依赖 IRL 自动调参，对复杂场景建议以专家初始化为基础进行微调
   - memory_tier: retrieval

5. **EVD-ED-correlation**
   - claim: 在基于概率策略选择的规划中，期望价值与期望距离呈高度相关，可同时作为训练和验证指标
   - evidence: Rosbach 等观察到 EVD 降低 63%-67% 与 ED 降低 44%-54% 的趋势高度相似
   - scope: 基于概率模型选择的规划评估
   - related_to: [[GeneralPurposePlanner]], [[MaximumEntropyIRL]]
   - confidence: medium
   - action: 在基于 MaxEnt IRL 的规划器训练中，同时监控 EVD 和 ED 以全面评估收敛
   - memory_tier: retrieval

### Episodic Memories（阅读洞察，2 条）

6. **planner-as-sampler-insight**
   - episode: 阅读方法部分时，意识到 Rosbach 等没有构建独立的采样模块来近似配分函数，而是直接复用规划器的搜索图
   - initial_understanding: 连续域 IRL 必须通过独立采样器（如 SMIRL）或解析近似（如 CIOC）来估计配分函数
   - confusion: 如果规划器自身搜索不充分，配分函数近似是否会严重偏差？
   - resolution_path: 通用型规划器的 GPU 并行搜索在每个周期生成约 4000 条策略，且剪枝操作确保多行为探索。虽然不如独立采样器可控，但对于特定规划器架构，这种"内生采样"更高效且无需额外计算开销。关键是规划器本身必须具有足够的状态空间覆盖能力
   - verified: high（实验结果支持）
   - future_hint: 评估是否可将此方法扩展至其他基于采样的规划器（如 SMIRL 的采样器本身）

7. **expert-init-vs-random-init-paradox**
   - episode: 对比图 4 的实验结果时发现，随机初始化在某些指标上改善幅度（73%）远超专家初始化（22%）
   - initial_understanding: 专家初始化应该为学习提供更好的起点，收敛更快，最终效果也更好
   - confusion: 为何专家初始化的绝对改善幅度反而更小？
   - resolution_path: 改善幅度是相对初始性能而言的。专家初始化初始性能已较好（起点高），绝对改善空间有限；随机初始化初始性能差（起点低），绝对改善空间大。此外，专家初始化在方差降低上表现更优（稳定性更好），而随机初始化在均值改善上表现更优。两种初始化策略各有优势
   - verified: high（图 3 和图 4 数据一致支持）
   - future_hint: 实际应用中可采用"专家初始化 + 持续更新"策略：以专家调参为起点，通过持续 IRL 微调适应特定场景

### Procedural Memories（程序规则，2 条）

8. **reuse-planner-for-IRL-sampling**
   - rule: 在使用基于搜索的自动驾驶规划器时，优先复用规划器自身的搜索图来近似 IRL 的配分函数，而非构建独立采样模块
   - applies_to: 基于 GPU 并行搜索或图搜索的自动驾驶规划器
   - why_it_works: 规划器的搜索过程本身就是对状态-动作空间的系统性探索，生成的策略集天然具有多样性和运动学可行性。复用搜索图避免了独立采样的计算开销和集成复杂性
   - when_not_to_use: 规划器搜索策略集规模过小（如 < 100 条策略）或覆盖不足时
   - confidence: high
   - action: 在通用型规划器、采样型规划器或图搜索规划器中集成 IRL 时，首先评估规划器自身搜索图是否足以近似配分函数

9. **initialization-strategy-selection**
   - rule: 在 IRL 奖励学习初始化策略选择中，若目标是最大化均值性能改善，从随机初始化开始；若目标是提高稳定性和降低方差，从专家初始化开始
   - applies_to: 所有需要从人类示范中学习奖励函数的 IRL 任务
   - why_it_works: 随机初始化起点低，改善空间大，适合探索更优解；专家初始化起点高，方差小，适合在已有基础上微调稳定
   - when_not_to_use: 有充足计算资源时，可同时训练两种初始化并集成
   - confidence: medium
   - action: 自动驾驶奖励调参中，对简单场景（如高速巡航）采用随机初始化，对复杂场景（如城市交叉口）采用专家初始化
